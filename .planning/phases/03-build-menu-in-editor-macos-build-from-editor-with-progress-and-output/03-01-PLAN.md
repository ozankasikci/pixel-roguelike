Create `src/editor/build/` directory and two files:

**EditorBuildSystem.h** must define:

1. `enum class BuildLineKind { Normal, Warning, Error, Progress }` — line classification.

2. `struct BuildOutputLog` — adapted from ImGui ExampleAppLog pattern:
   - `ImGuiTextBuffer buf` — raw text buffer
   - `ImVector<int> lineOffsets` — byte offsets for each line start (init with push_back(0))
   - `ImVector<BuildLineKind> lineKinds` — per-line classification
   - `bool autoScroll = true`
   - `int firstErrorLine = -1` — index of first error line for jump-to-error (D-12)
   - `bool scrollToError = false` — flag to trigger scroll-to-error on next frame
   - `void clear()` — resets buf, lineOffsets (re-push 0), lineKinds, firstErrorLine
   - `void addLine(const char* line, BuildLineKind kind)` — appends line + newline to buf, records offset and kind; if kind == Error and firstErrorLine == -1, sets firstErrorLine
   - `int lineCount() const` — returns lineOffsets.Size (number of lines)

3. `struct EditorBuildConfig` — persistent build preferences:
   - `std::string buildDir = "build"` — configurable build directory (D-02)
   - `std::string buildConfig = "Debug"` — Debug/Release/RelWithDebInfo (D-04)

4. `struct EditorBuildState` — runtime state of a build:
   - `pid_t pid = -1` — PID of the cmake child process
   - `int pipeFd = -1` — read end of the pipe
   - `bool running = false` — whether a build is in progress
   - `bool runAfterBuild = false` — true when triggered by "Build and Run" (D-17)
   - `int exitCode = -1` — exit code after process completes
   - `float progressPct = 0.0f` — last parsed percentage (D-10)
   - `std::string currentScenePath` — scene path to pass via --scene when running (D-18)
   - `std::mutex queueMutex`
   - `std::vector<std::string> pendingLines` — reader thread enqueues here
   - `std::thread readerThread`
   - `std::atomic<bool> readerDone{false}`

5. Free functions (declared in header, implemented in .cpp):
   - `BuildLineKind classifyLine(const std::string& line, float& progressOut)` — scans for `[ XX%]` via sscanf (both `"[ %d%%]"` and `"[%d%%]"` patterns), scans for `error:` and `warning:` (case-insensitive via tolower transform). Returns appropriate BuildLineKind. Sets progressOut if Progress.
   - `void startBuild(EditorBuildState& state, const EditorBuildConfig& config, const std::string& target)` — Creates pipe via `pipe()`. Forks via `fork()`. In child: calls `setpgid(0, 0)` to create new process group, `dup2(pipefd[1], STDOUT_FILENO)`, `dup2(pipefd[1], STDERR_FILENO)`, closes both pipe ends, then `execvp("cmake", argv)` where argv is `{"cmake", "--build", buildDir.c_str(), "--target", target.c_str(), "--parallel", nullptr}`. Calls `_exit(1)` if exec fails. In parent: closes write end, sets read end non-blocking via `fcntl(pipefd[0], F_SETFL, O_NONBLOCK)`, stores pid/pipeFd, sets running=true, resets progressPct=0, readerDone=false, spawns readerThread.
   - `void startConfigure(EditorBuildState& state, const EditorBuildConfig& config)` — Same fork/exec pattern but runs `cmake -S . -B {buildDir} -DCMAKE_BUILD_TYPE={buildConfig}` (for auto-configure D-03 and config switch D-04). The argv is `{"cmake", "-S", ".", "-B", buildDir.c_str(), "-DCMAKE_BUILD_TYPE=...", nullptr}`. The `-DCMAKE_BUILD_TYPE=X` must be a single string constructed as `std::string cmakeBuildType = "-DCMAKE_BUILD_TYPE=" + config.buildConfig;`.
   - `void cancelBuild(EditorBuildState& state)` — Sends `kill(-state.pid, SIGTERM)` (negative PID for process group) per D-14. Does NOT set running=false here (waitpid in pollBuild handles that).
   - `void pollBuild(EditorBuildState& state, BuildOutputLog& log)` — Drains pendingLines via lock+swap pattern. For each line calls classifyLine, updates progressPct if progress detected, calls log.addLine. Then calls `waitpid(state.pid, &status, WNOHANG)`. If process exited: sets running=false, extracts exit code via `WIFEXITED/WEXITSTATUS`, sets pid=-1, joins readerThread, closes pipeFd. If exitCode != 0, sets log.scrollToError = true.
   - `bool needsConfigure(const EditorBuildConfig& config)` — Returns true if `{config.buildDir}/CMakeCache.txt` does not exist (uses `std::filesystem::exists`). Per D-03.
   - `std::string gameBinaryPath(const EditorBuildConfig& config)` — Returns `config.buildDir + "/apps/runtime/pixel-roguelike"`. Per verified binary output path.
   - `void launchGame(const EditorBuildConfig& config, const std::string& scenePath)` — Fork+exec the game binary. In child: `setpgid(0, 0)`, then `execvp(binaryPath, argv)` where argv includes `--scene scenePath` if scenePath is not empty (D-18). Parent does NOT wait (fire-and-forget). Per D-17.
   - `void loadBuildConfig(EditorBuildConfig& config, const std::string& path)` — Reads an INI file line by line, parsing `build_dir=X` and `build_config=X` via sscanf. Follows the same pattern as `loadWindowGeometry()` in main.cpp.
   - `void saveBuildConfig(const EditorBuildConfig& config, const std::string& path)` — Writes `build_dir=X\nbuild_config=X\n` to the file. Simple ofstream write.

The reader thread function (can be a static helper or lambda in startBuild):
- Signature: takes `int fd`, `std::mutex& mtx`, `std::vector<std::string>& queue`, `std::atomic<bool>& done`
- Reads from fd in a loop: `read(fd, buf, 4096)`. On n > 0: append to partial string, extract complete lines (split on '\n'), lock mutex, push each complete line to queue. On EAGAIN: sleep 16ms. On n == 0 or error: break.
- Sets done = true before returning.

IMPORTANT implementation notes:
- All `execvp` calls need a `const char* argv[]` array. Since execvp takes `char* const argv[]`, the strings must be kept alive. Use `std::vector<std::string>` for owned strings and build a `std::vector<const char*>` for the argv array, terminated with nullptr.
- The `startConfigure` function must construct the `-DCMAKE_BUILD_TYPE=Debug` string (or Release/RelWithDebInfo) as a single concatenated string stored in a vector element before building the argv pointer array.
- Use `spdlog::info` to log build start, build cancel, build complete (exit code), and game launch events.
- Do NOT use `popen()` (no PID), `std::system()` (blocking), or two separate pipes (deadlock risk).
