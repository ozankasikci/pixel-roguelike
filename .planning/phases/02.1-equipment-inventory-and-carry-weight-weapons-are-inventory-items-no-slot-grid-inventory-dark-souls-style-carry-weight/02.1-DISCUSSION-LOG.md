## Inventory Scope

| Option | Description | Selected |
|--------|-------------|----------|
| Weapons and normal carryable loot | Weapons and normal carryable loot go into inventory, but key/progression items stay separate and weightless | |
| Everything in one inventory | Everything goes into one inventory and everything can have weight | |
| Weapons only for this phase | Weapons only for this phase; other item categories come later | ✓ |

**User's choice:** Weapons only for this phase.
**Notes:** The user wants weapons to be proper inventory items now, while broader item categories remain future work.

---

## Equipment Model

| Option | Description | Selected |
|--------|-------------|----------|
| One active weapon slot | Simplest, but less Souls-like | |
| Dedicated melee/ranged equipment slots | Inventory equips into future combat-type slots | |
| Multi-slot Souls-like reserve model | Multiple hand-oriented equipment slots, closer to Souls complexity | ✓ |

**User's choice:** Multi-slot Souls-like reserve model.
**Notes:** Follow-up clarified the desired layout as one right hand, one left hand, plus support for two-handed weapons.

---

## Carry Weight Rules

| Option | Description | Selected |
|--------|-------------|----------|
| True equip load | Equipped gear determines burden; bag contents do not slow the player | ✓ |
| Full carry weight | Every carried item counts toward burden and capacity | |
| Hybrid | Carried items count toward capacity, equipped items drive movement burden tiers | |

**User's choice:** True equip load.
**Notes:** Unequipped weapons should have no burden.

---

## Inventory Presentation

| Option | Description | Selected |
|--------|-------------|----------|
| Paused inventory screen | Category list, item list, details pane, burden meter in a minimal 1-bit style | ✓ |
| Compact live overlay | Inventory shown while the world keeps running | |
| No full screen yet | Only lightweight prompts and equip actions for now | |

**User's choice:** Paused inventory screen.
**Notes:** The user wants the system to feel Dark Souls-like rather than like a moment-to-moment overlay.

---

## Slot Layout and Access

| Option | Description | Selected |
|--------|-------------|----------|
| Switch during gameplay | Cycle equipped slots during active play | |
| Switch from inventory only | Changes happen only while paused in inventory | ✓ |
| Other | Custom behavior described in follow-up | |

**User's choice:** Switch from inventory only.
**Notes:** Inventory access should pause the game, and the default key should be `I`. `E` remains the separate interaction key already used by world prompts.
