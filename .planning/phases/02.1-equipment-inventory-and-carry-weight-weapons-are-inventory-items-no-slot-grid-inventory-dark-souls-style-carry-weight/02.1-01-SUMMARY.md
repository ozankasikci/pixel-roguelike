# Phase 02.1 Plan 01: Weapon Inventory Data Contracts Summary

**Weapon inventory metadata, explicit owned-vs-equipped hand state, and test-backed two-handed burden rules for the upcoming paused inventory UI**

## Accomplishments

- Extended weapon defs and parsing with handedness, equip weight, category, and description metadata needed by the inventory screen
- Added a baseline two-handed authored weapon (`brigand_axe`) so the phase has real content for equipment and burden behavior
- Replaced the old boolean loadout concept with explicit owned weapon inventory plus left-hand/right-hand equipment state in `RunSession`
- Added a canonical `EquipmentState` helper layer for equip, unequip, effective occupancy, and burden computation
- Added regression coverage proving the equipped-only burden rule and two-handed conflict-clearing behavior

## Decisions Made

- Two-handed occupancy is represented by the same weapon id in both hands, with burden counted once from the effective view
- A one-handed equip request always clears any active two-handed occupancy first
- Starting owned weapons are seeded directly in `RunSession` for this phase: `old_dagger` and `brigand_axe`
- The asset parser remains whitespace-tokenized for now, so authored inventory-facing text stays underscore-safe
