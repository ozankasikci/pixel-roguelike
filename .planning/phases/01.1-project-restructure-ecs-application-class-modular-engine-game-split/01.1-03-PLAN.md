    **src/game/systems/CameraSystem.cpp**:
    Extracts the camera logic from main.cpp lines 88-139. Each frame:
    1. Query registry for entities with TransformComponent + CameraComponent (there will be exactly one -- the player camera).
    2. Read key state from InputSystem (GLFW_KEY_W/A/S/D for movement, arrow keys for look).
    3. Check input_.wantsCaptureMouse() -- if true, skip WASD movement (same as current ImGui gate on line 99).
    4. Update yaw/pitch from arrow keys (lookSpeed = 60.0f * deltaTime, same as main.cpp line 115-116).
    5. Clamp pitch to [-89, 89] (same as main.cpp line 123-124).
    6. Compute forward vector from yaw/pitch (same math as main.cpp lines 89-93 and 126-130).
    7. Compute right vector as cross(forward, up).
    8. Apply WASD movement: position += forward/right * moveSpeed * deltaTime.
    9. Store forward and right into CameraComponent.
    10. Compute viewMatrix = glm::lookAt(position, position + forward, up) and store in CameraComponent.
    11. Compute projectionMatrix = glm::perspective(radians(fov), aspect, near, far) and store in CameraComponent.

    For aspect ratio: the CameraSystem needs the FBO dimensions. It can get these from the RenderSystem via an event, or it can use the window dimensions. For now, use window().width() / window().height() from the Application -- this matches the existing behavior where aspect is computed from sceneFBO dimensions (which defaults to 720p matching window size). The RenderSystem will use CameraComponent.projectionMatrix which has the correct aspect.

    IMPORTANT: The exact same math must be used as main.cpp to guarantee identical camera behavior. Copy the formulas exactly: cos(radians(yaw)) * cos(radians(pitch)), sin(radians(pitch)), sin(radians(yaw)) * cos(radians(pitch)).
