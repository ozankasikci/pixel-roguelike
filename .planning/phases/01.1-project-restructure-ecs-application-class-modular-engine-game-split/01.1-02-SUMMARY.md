# Phase 01.1 Plan 02: Engine Core Infrastructure and ECS Components Summary

**Application class with game loop and system lifecycle, EventBus, Time, and four ECS component types (Transform, Mesh, Light, Camera) -- architectural backbone for all plan 03 and 04 systems**

## Accomplishments

- Application class owns Window, entt::registry, EventBus, Time, and system vector; addSystem<T>() template registers systems
- System base class establishes init/update/shutdown pure-virtual contract for all future game systems
- EventBus provides type-safe publish/subscribe with no boilerplate using std::type_index + std::any
- Time encapsulates deltaTime logic from main.cpp with glfwGetTime(), preserving identical frame-timing behavior
- Four ECS components defined matching existing data structures (PointLight fields, camera params from main.cpp)
- CMakeLists.txt updated to compile Application.cpp and Time.cpp into engine_core and link EnTT::EnTT
- Existing binary unchanged; screenshot output identical (main.cpp not modified per plan)

## Files Created/Modified

- `src/engine/core/Application.h` - Application class: game loop, addSystem<T>(), registry/eventBus/window/time accessors

## Decisions Made

- Time::update() preserves the exact logic from main.cpp lines 75-86 to avoid any deltaTime behavior change
- MeshComponent::modelOverride bool flag allows CathedralScene arch segments to bypass euler decomposition
- CameraComponent initial values copied from main.cpp hardcoded values for seamless plan 04 migration
- Application::run() reverse-order shutdown is intentional for RAII-correct teardown
