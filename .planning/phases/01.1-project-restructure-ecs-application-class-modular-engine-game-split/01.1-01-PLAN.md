  <action>
    Per D-16, D-17: Restructure CMakeLists.txt to build static libraries per engine module.
    Per D-07: Add EnTT v3.16.0 via FetchContent.

    Replace the monolithic add_executable source list with per-module static libraries:

    1. Add EnTT FetchContent (after existing GLAD block):
    ```cmake
    # EnTT v3.16.0 - Entity Component System
    FetchContent_Declare(entt
        GIT_REPOSITORY https://github.com/skypjack/entt.git
        GIT_TAG        v3.16.0
        GIT_PROGRESS   TRUE
    )
    FetchContent_MakeAvailable(entt)
    ```

    2. Create engine_core static library:
    ```cmake
    add_library(engine_core STATIC
        src/engine/core/Window.cpp
    )
    target_include_directories(engine_core PUBLIC ${CMAKE_SOURCE_DIR}/src)
    target_link_libraries(engine_core PUBLIC glad_gl glfw spdlog::spdlog)
    ```

    3. Create engine_rendering static library:
    ```cmake
    add_library(engine_rendering STATIC
        src/engine/rendering/Shader.cpp
        src/engine/rendering/Framebuffer.cpp
        src/engine/rendering/DitherPass.cpp
        src/engine/rendering/Mesh.cpp
        src/engine/rendering/Renderer.cpp
    )
    target_include_directories(engine_rendering PUBLIC ${CMAKE_SOURCE_DIR}/src)
    target_link_libraries(engine_rendering PUBLIC engine_core glm::glm)
    ```

    4. Create engine_ui static library:
    ```cmake
    add_library(engine_ui STATIC
        src/engine/ui/ImGuiLayer.cpp
        src/engine/ui/Screenshot.cpp
    )
    target_include_directories(engine_ui PUBLIC
        ${CMAKE_SOURCE_DIR}/src
        ${CMAKE_SOURCE_DIR}/external
    )
    target_link_libraries(engine_ui PUBLIC engine_rendering imgui)
    ```

    5. Create game static library:
    ```cmake
    add_library(game STATIC
        src/game/scenes/CathedralScene.cpp
    )
    target_include_directories(game PUBLIC ${CMAKE_SOURCE_DIR}/src)
    target_link_libraries(game PUBLIC engine_rendering EnTT::EnTT)
    ```

    6. Update the executable to link against the libraries instead of listing source files:
    ```cmake
    add_executable(pixel-roguelike src/main.cpp)
    target_include_directories(pixel-roguelike PRIVATE ${CMAKE_SOURCE_DIR}/src ${CMAKE_SOURCE_DIR}/external)
    target_link_libraries(pixel-roguelike PRIVATE
        engine_core
        engine_rendering
        engine_ui
        game
        EnTT::EnTT
    )
    ```

    7. Keep the Apple-specific block for the executable target.

    8. Keep the existing GLAD, ImGui, and system package blocks unchanged.

    Per D-17: Each lib declares its own include dirs and dependencies. engine_rendering depends on engine_core (for glad/glfw). engine_ui depends on engine_rendering (for DitherParams, Renderer types). game depends on engine_rendering.

    IMPORTANT: The target_include_directories on each library must use PUBLIC with ${CMAKE_SOURCE_DIR}/src so that consumers automatically get the include path. This avoids needing PRIVATE includes on every target.
  </action>
