    Create four new engine core files. These establish the contracts that systems will implement against.

    **src/engine/core/System.h** (per D-05):
    Pure abstract base class. Systems register with Application and receive lifecycle calls.
    ```cpp
    #pragma once
    #include <entt/entt.hpp>

    class Application; // forward declare
    class EventBus;    // forward declare
    class Time;        // forward declare

    class System {
    public:
        virtual ~System() = default;
        virtual void init(Application& app) = 0;
        virtual void update(Application& app, float deltaTime) = 0;
        virtual void shutdown() = 0;
    };
    ```
    Systems receive Application& so they can access registry(), eventBus(), window() etc. per D-06.

    **src/engine/core/EventBus.h** (per D-11, D-12):
    Template-based type-safe publish/subscribe. Header-only.
    ```cpp
    #pragma once
    #include <functional>
    #include <unordered_map>
    #include <vector>
    #include <typeindex>
    #include <any>

    class EventBus {
    public:
        template<typename EventType>
        void subscribe(std::function<void(const EventType&)> handler) {
            auto key = std::type_index(typeid(EventType));
            subscribers_[key].push_back([handler](const std::any& event) {
                handler(std::any_cast<const EventType&>(event));
            });
        }

        template<typename EventType>
        void publish(const EventType& event) {
            auto key = std::type_index(typeid(EventType));
            auto it = subscribers_.find(key);
            if (it != subscribers_.end()) {
                for (auto& handler : it->second) {
                    handler(event);
                }
            }
        }

    private:
        std::unordered_map<std::type_index, std::vector<std::function<void(const std::any&)>>> subscribers_;
    };
    ```
    This is intentionally simple. No unsubscribe needed yet -- systems live for the full app lifetime.

    **src/engine/core/Time.h / Time.cpp**:
    Frame timing utility extracted from the deltaTime logic currently in main.cpp.
    ```cpp
    // Time.h
    #pragma once

    class Time {
    public:
        void update();       // call once per frame
        float deltaTime() const { return deltaTime_; }
        float elapsed() const { return elapsed_; }

    private:
        float deltaTime_ = 0.0f;
        float elapsed_ = 0.0f;
        float lastTime_ = 0.0f;
        bool firstFrame_ = true;
    };
    ```
    Time.cpp: uses glfwGetTime(). On first frame, sets lastTime_ = glfwGetTime(), deltaTime_ = 0.0001f. On subsequent frames: currentTime - lastTime_, clamp minimum to 0.0001f. This mirrors the exact logic from main.cpp lines 75-86.

    **src/engine/core/Application.h / Application.cpp** (per D-04, D-05, D-06):
    Owns Window, ECS registry, EventBus, Time, and a vector of System unique_ptrs.
    ```cpp
    // Application.h
    #pragma once
    #include <memory>
    #include <vector>
    #include <entt/entt.hpp>
    #include "engine/core/Window.h"
    #include "engine/core/EventBus.h"
    #include "engine/core/Time.h"

    class System;

    class Application {
    public:
        Application(int width, int height, const char* title);
        ~Application();

        // Non-copyable
        Application(const Application&) = delete;
        Application& operator=(const Application&) = delete;

        void run();  // main game loop

        // System management (per D-05)
        template<typename T, typename... Args>
        T& addSystem(Args&&... args) {
            auto system = std::make_unique<T>(std::forward<Args>(args)...);
            T& ref = *system;
            systems_.push_back(std::move(system));
            return ref;
        }

        // Accessors (per D-06)
        Window& window() { return window_; }
        entt::registry& registry() { return registry_; }
        EventBus& eventBus() { return eventBus_; }
        const Time& time() const { return time_; }
        float deltaTime() const { return time_.deltaTime(); }

    private:
        Window window_;
        entt::registry registry_;
        EventBus eventBus_;
        Time time_;
        std::vector<std::unique_ptr<System>> systems_;
        bool running_ = false;
    };
    ```

    Application.cpp: Constructor creates window. run() implements the game loop:
    1. Call init() on all systems
    2. Loop while !window_.shouldClose():
       a. window_.pollEvents()
       b. time_.update()
       c. Call update(app, deltaTime) on all systems in order
       d. window_.swapBuffers()
    3. Call shutdown() on all systems in reverse order

    The run() loop is the replacement for the current main.cpp while loop. But main.cpp is NOT changed in this plan -- that happens in Plan 04.

    IMPORTANT: Do NOT modify src/main.cpp in this plan. The new Application class is created alongside the existing code. Integration happens in Plan 04.

    **src/game/components/TransformComponent.h** (per D-10):
    ```cpp
    #pragma once
    #include <glm/glm.hpp>
    #include <glm/gtc/matrix_transform.hpp>

    struct TransformComponent {
        glm::vec3 position{0.0f};
        glm::vec3 rotation{0.0f};  // euler angles in degrees
        glm::vec3 scale{1.0f};

        glm::mat4 modelMatrix() const {
            glm::mat4 m = glm::translate(glm::mat4(1.0f), position);
            m = glm::rotate(m, glm::radians(rotation.x), glm::vec3(1, 0, 0));
            m = glm::rotate(m, glm::radians(rotation.y), glm::vec3(0, 1, 0));
            m = glm::rotate(m, glm::radians(rotation.z), glm::vec3(0, 0, 1));
            m = glm::scale(m, scale);
            return m;
        }
    };
    ```
    NOTE: CathedralScene currently uses pre-computed model matrices (translate+rotate+scale inline). The TransformComponent provides a standard way to compute these, but CathedralScene's refactor in Plan 04 may still store pre-computed matrices for geometry that doesn't use standard euler rotation (arch segments). That's fine -- TransformComponent is the general case.
